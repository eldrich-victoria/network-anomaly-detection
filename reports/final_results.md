# Final Results Report: ShieldNet AI Network Anomaly Detection System

## 1. Executive Summary
This report presents the final results of **ShieldNet AI**, an unsupervised machine learning system designed to identify cyber security anomalies in network flow traffic using the UNSW-NB15 benchmark dataset. The system trains and compares four distinct anomaly detection architectures (Isolation Forest, Local Outlier Factor, One-Class SVM, and PyTorch Autoencoder) to evaluate their capabilities in high-throughput network monitoring.

---

## 2. Dataset Overview: UNSW-NB15
The UNSW-NB15 dataset is an industry-standard benchmark dataset for evaluating network intrusion detection systems, generated by the Cyber Range Lab of the Australian Centre for Cyber Security (ACCS). 
* **Training Set Size**: 175,341 rows
* **Test Set Size**: 82,332 rows
* **Total Features**: 45 columns (including flow duration, transaction rates, protocols, service types, state metrics, and packet count statistics)
* **Target Distribution (Test Set)**:
  * **Normal (Clean)**: 37,000 flows (44.94%)
  * **Anomaly (Attack)**: 45,332 flows (55.06%)
  * **Attack Categories**: Fuzzers, Analysis, Backdoor, DoS, Exploits, Generic, Reconnaissance, Shellcode, Worms.

---

## 3. Data Preprocessing Pipeline
To feed raw network records into machine learning models, we implement a robust preprocessing pipeline:
1. **Cleaning**: Missing nominal attributes (such as `proto`, `service`, `state`) are filled with `'unknown'` and numeric features filled with `0.0`.
2. **Label Encoding**: Nominal columns are transformed using a custom `RobustLabelEncoder` that maps unseen values in production to an `'unknown'` bin, preventing runtime crashes.
3. **Scaling**: Numerical features are normalized using a `StandardScaler` to ensure features with large scales (such as `sbytes` and `dbytes`) do not dominate distance/error metrics.
4. **Feature Selection**: The 43 core network features are selected, excluding non-predictive identifiers like `id`.

## 4. Implemented Model Architectures

### A. Isolation Forest (Ensemble Learning)
* **Paradigm**: Decision Tree Ensemble
* **Description**: Isolates anomalies by randomly selecting a feature and split value. Because anomalies require fewer splits to partition, they lie closer to the roots of trees.
* **Tuning Parameters**: `n_estimators=100`, `contamination=0.1`

### B. Local Outlier Factor (Density-based)
* **Paradigm**: Distance & Density Deviation
* **Description**: Measures the local density deviation of a given data point relative to its neighbors. Low density relative to neighbors indicates an outlier.
* **Tuning Parameters**: `n_neighbors=20`, `novelty=True` (for out-of-sample prediction)

### C. One-Class SVM (Boundary-based)
* **Paradigm**: Support Vector Hypersphere
* **Description**: Maps normal flows into a high-dimensional space and fits a tight boundary hypersphere. Flows falling outside the hypersphere are flagged as anomalies.
* **Tuning Parameters**: `kernel='rbf'`, `nu=0.05`

### D. PyTorch Autoencoder (Deep Reconstruction Learning)
* **Paradigm**: Neural Network Bottleneck
* **Description**: A multi-layer perceptron trained only on normal traffic records to reconstruct inputs. The bottleneck forces the network to learn structural correlations. Anomalous flows result in high reconstruction Mean Squared Error (MSE), crossing the outlier threshold.
* **Network Topology**:
  * Input layer: 42 dimensions
  * Encoder: 42 -> 32 -> 16 -> 8
  * Decoder: 8 -> 16 -> 32 -> 42
  * Loss: Mean Squared Error (MSE)
  * Optimizer: Adam

## 5. Real-Time Prediction and Alert Pipeline
The prediction pipeline (`predict.py`) exposes an `AnomalyPredictor` interface. When a new batch of network flows is submitted:
1. The batch is preprocessed using the trained scaler and encoders.
2. The selected model runs inference.
3. For detected anomalies, the `AlertSystem` calculates severity levels (Low, Medium, High, Critical) based on how far the anomaly score exceeds the decision threshold.
4. Alerts are logged dynamically to `alerts.csv` with core context attributes (Timestamp, Model, Anomaly Score, Protocol, Service, State, Duration, Bytes, and Rate) for security analyst response.
