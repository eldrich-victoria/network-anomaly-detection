# AI-Powered Network Anomaly Detection System - Project Summary

This project implements an end-to-end network anomaly detection system using the UNSW-NB15 dataset. It includes data loaders, modular preprocessing, exploratory data analysis, four trained models (Isolation Forest, Local Outlier Factor, One-Class SVM, and PyTorch Deep Autoencoder), evaluations, prediction pipelines, threat alerting, and a Streamlit dashboard.

---

## 1. Project Background and Objective

Network intrusion and anomaly detection are essential pillars of modern cybersecurity. Rather than relying solely on signature-based detection (which misses novel, zero-day attacks), modern Security Operations Centers (SOCs) deploy machine learning and anomaly detection models. 

This project implements and compares four different anomaly detection techniques to identify outliers in netflow traffic records, providing cybersecurity administrators with a real-time web console to analyze network flows and respond to incidents.

---

## 2. Dataset and Preprocessing

### UNSW-NB15 Dataset
The UNSW-NB15 dataset contains modern network traffic flows generated by the Cyber Range Lab of the Australian Centre for Cyber Security. It contains 45 features representing flow records (duration, packet counts, bytes transfer, protocols, services, TTLs, and connection states). The dataset contains both clean normal traffic and 9 categories of attacks.

### Data Cleaning and Transformation
Our preprocessing module executes the following:
1. **Placeholder Cleaning**: Unifies string missing markers (such as `'-'` in the `service` column) into a standard `'unknown'` label.
2. **Robust Encoding**: Converts categorical variables (`proto`, `service`, `state`) using a custom `RobustLabelEncoder` that maps unseen values to an `'unknown'` bin, preventing runtime prediction failures.
3. **Feature Scaling**: Scales 39 numerical features using `StandardScaler` to bring them to a zero-mean and unit-variance distribution.
4. **Duplicates Management**: Drops duplicate records in the training set to prevent model overfitting.
5. **Serialization**: Saves scaler and encoders to the `models/` directory for use in real-time prediction.

## 3. Anomaly Detection Models

Four anomaly detection techniques were trained and configured:

1. **Isolation Forest**
   - **Type**: Unsupervised tree-based ensemble.
   - **Mechanism**: Splits features randomly. Anomalies require fewer splits (shallower path length) to isolate compared to normal data.
   - **Configuration**: Capped contamination at `0.49`, RBF-like partition search, 100 estimators.

2. **Local Outlier Factor (LOF)**
   - **Type**: Density-based clustering outlier detection.
   - **Mechanism**: Measures local density deviation of points relative to neighbors. Outliers have a significantly lower density than their K-neighbors.
   - **Configuration**: Trained with `novelty=True` on a subsample of 5,000 normal traffic flows to optimize memory footprints and avoid system OOM errors.

3. **One-Class SVM**
   - **Type**: Boundary-based classification.
   - **Mechanism**: Maps data to a high-dimensional space and fits a tight hypersphere boundary around the normal class.
   - **Configuration**: Trained on 5,000 normal traffic samples with an RBF kernel and `nu=0.05` to keep training extremely efficient.

4. **Deep Autoencoder**
   - **Type**: Deep Reconstruction Learning (PyTorch Fallback).
   - **Mechanism**: Trained on normal data only (`label == 0`). Learns to compress (bottleneck) and reconstruct normal traffic flows. During test/predict, unseen anomaly signatures yield high reconstruction error (Mean Squared Error).
   - **Architecture**: Input (42) -> Dense(128, ReLU) -> Dense(64, ReLU) -> Dense(32, ReLU) -> Dense(64, ReLU) -> Dense(128, ReLU) -> Output (42, Linear).
   - **Threshold**: Set at the 95th percentile of normal training errors (`0.171268`).

## 4. Quantitative Results & Evaluation

The models were evaluated on the independent test dataset (82,332 rows, including 37,000 normal and 45,332 anomalies). The results are summarized below:

| Model | Accuracy | Precision | Recall | F1-Score |
| --- | --- | --- | --- | --- |
| **Local Outlier Factor** | **80.64%** | **92.12%** | 70.91% | **80.14%** |
| **Deep Autoencoder** | 69.26% | 86.44% | 52.39% | 65.24% |
| **One-Class SVM** | 63.17% | 90.29% | **37.11%** | 52.60% |
| **Isolation Forest** | 36.88% | 42.20% | 39.60% | 40.86% |

### Key Findings
- **Local Outlier Factor (LOF)** achieved the highest F1-score of **80.14%** and accuracy of **80.64%**, making it the top performing model for identifying outliers in the test set.
- **Deep Autoencoder** achieved a solid precision of **86.44%** with an F1-score of **65.24%**, showing high reliability (few false alerts).
- **One-Class SVM** has a high precision of **90.29%** but lower recall, showing that it identifies clean boundaries but misses some anomalies.
- **Isolation Forest** had lower performance due to the highly skewed features and complex multivariate boundaries of the UNSW-NB15 dataset.

## 5. Future Work & Enhancements

1. **Feature Importance & Selection**: Use tree-based classifiers or Mutual Information to identify and retain only the top 15-20 most informative features, reducing the dimensionality and speeding up training.
2. **Semi-Supervised Autoencoder Tuning**: Tune the bottleneck dimension (e.g. reducing the latent space to 16 or 8 neurons) to force a tighter compression layer, which can improve anomaly reconstruction errors.
3. **Ensemble Voting**: Build a majority-voting or weighted ensemble model combining LOF and Autoencoder predictions to achieve higher overall accuracy and F1-score.
4. **Kafka Integration**: Hook the prediction pipeline to a real-time message stream like Apache Kafka for live, sub-millisecond netflow monitoring.
